# Adding New Chart Pages - Implementation Guide

This guide outlines the process for adding new chart showcase pages to the Warphead Charts section, following established patterns and conventions extracted from the barchart and linechart implementations.

## Overview

When adding a new chart page, you need to:
1. Create the chart page directory and component (`+page.svelte`)
2. Add a navigation card to the index page (`+page.svelte`)
3. Follow the established structure, patterns, and styling conventions
4. Include responsive resize observers for all charts
5. Provide multiple chart examples showcasing different use cases
6. Add a help section with documentation links

---

## File Structure

### Chart Page Location

### Required Files
- `{chartname}/+page.svelte` - The chart showcase page component

### 3. Responsive Chart Pattern

#### State Variables Pattern
- [ ] **Container widths**: Use numbered state variables (`containerWidth1`, `containerWidth2`, etc.)
- [ ] **Container elements**: Use numbered element references (`containerEl1`, `containerEl2`, etc.)
- [ ] **Naming**: Number sequentially starting from 1

```svelte
let containerWidth1 = $state(0);
let containerWidth2 = $state(0);
// ... continue for each chart

let containerEl1 = $state<HTMLDivElement>();
let containerEl2 = $state<HTMLDivElement>();
// ... continue for each chart
```

#### Resize Observer Function
- [ ] **Standard function**: Use the exact `setupResizeObserver` function pattern
- [ ] **Initial measurement**: Call `updateSize()` immediately
- [ ] **RequestAnimationFrame**: Use `requestAnimationFrame` to ensure DOM is ready
- [ ] **ResizeObserver**: Create and observe the element

```svelte
function setupResizeObserver(el: HTMLDivElement | undefined, setWidth: (w: number) => void) {
	if (!el) return;
	
	const updateSize = () => {
		if (el) {
			const width = el.clientWidth || el.offsetWidth;
			if (width > 0) {
				setWidth(width);
			}
		}
	};
	
	// Initial measurement
	updateSize();
	
	// Use requestAnimationFrame to ensure DOM is ready
	requestAnimationFrame(() => {
		updateSize();
		const resizeObserver = new ResizeObserver(updateSize);
		resizeObserver.observe(el);
	});
}
```

#### Effect Setup Pattern
- [ ] **Use $effect**: Set up all resize observers in a single `$effect` block
- [ ] **One per chart**: Call `setupResizeObserver` for each chart container

```svelte
$effect(() => {
	setupResizeObserver(containerEl1, (w) => containerWidth1 = w);
	setupResizeObserver(containerEl2, (w) => containerWidth2 = w);
	setupResizeObserver(containerEl3, (w) => containerWidth3 = w);
	// ... continue for each chart
});
```

#### PageLayout Wrapper
- [ ] **Always use PageLayout**: Wrap entire content in `<PageLayout>` component
- [ ] **Max width container**: Use `<div class="max-w-full mx-auto">` as immediate child

- [ ] **Consistent gap**: Always use `gap-6`

### 6. Plot Component Configuration

#### Standard Plot Props
- [ ] **Width**: Use `width={containerWidthN}` (from state)
- [ ] **Height**: Always use `height={400}`
- [ ] **Margins**: Use consistent margin values:
  - `marginLeft={60}` (or `120` for horizontal charts with long labels)
  - `marginBottom={60}`
  - `marginTop={20}`
  - `marginRight={20}`

#### Axis Configuration
- [ ] **X-axis**: Use `x={{ label: 'Label Name' }}` or `x={{ label: 'Label', tickFormat: formatFunction }}`
- [ ] **Y-axis**: Use `y={{ grid: true, label: 'Label Name' }}`
- [ ] **Grid lines**: Always enable on Y-axis with `grid: true`

```svelte
<Plot
	width={containerWidth1}
	height={400}
	marginLeft={60}
	marginBottom={60}
	marginTop={20}
	marginRight={20}
	x={{ label: 'Category' }}
	y={{ grid: true, label: 'Value' }}
>
	<!-- Chart mark components -->
</Plot>
```

#### Custom Tick Formatting
- [ ] **Format function**: Define helper function for custom labels
- [ ] **Pattern**: Use data lookup when labels are stored separately

```svelte
// Format function for time series
function formatMonth(d: number) {
	const item = timeSeriesData.find(item => item.date === d);
	return item?.label || String(d);
}

// Usage in Plot
x={{ label: 'Month', tickFormat: formatMonth }}
```

## Common Patterns by Chart Type

### Bar Charts (BarY, BarX)
- Vertical bars: `BarY` with `x="category" y="value"`
- Horizontal bars: `BarX` with `x="value" y="category"` and `marginLeft={120}`
- Stacked: Use `fill="series"` with restructured data
- Grouped: Multiple `BarY` components with different `y` props

### Line Charts (LineY)
- Basic: `LineY` with `stroke` and `strokeWidth` props
- Multi-series: Multiple `LineY` components
- Time series: Use `tickFormat` for custom labels
- Thick lines: Increase `strokeWidth` (e.g., `strokeWidth={4}`)

### Area Charts (AreaY)
- Similar to LineY but with filled areas
- Use `fill` and `fillOpacity` props
- Can be stacked with `fill="series"`

### Scatter Plots (Dot)
- Use `Dot` component
- May need `r` prop for point size
- Can use `fill` or `stroke` for colors
